Core Rules — What You Must Know

BFS — Breadth-First Search

▸Data structure: FIFO Queue
▸Add children to BACK of queue
▸Expand shallowest node first
▸Goal test on GENERATION
▸Optimal only if path costs are non-decreasing

DFS — Depth-First Search

▸Data structure: LIFO Stack
▸Add children to FRONT (top) of frontier
▸Expand deepest node first
▸Goal test on EXPANSION
▸NOT optimal — finds first path found
▸Frontier shown as [top, …, bottom]

UCS — Uniform Cost Search

▸Data structure: Priority Queue
▸Ordered by g(n) = total path cost
▸Expand MINIMUM cost node
▸Goal test on EXPANSION (pop)
▸Always optimal
▸Update frontier if a lower-cost path is found
